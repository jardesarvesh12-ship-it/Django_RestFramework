Djano Serializers Fields and Core Arguments------->

>>> Fields (fields in serializer) >>> serializers.py
* used to convert model object into python dict, and convert python dict to model object.


>>> In the Django REST Framework (DRF), core arguments are specific parameters used when defining serializer fields to control how data is validated,
transformed, and displayed.

Serializer Arguments 
1. many (bool) — Controls list serialization
2. read_only (bool) — Prevents write operations (e.g., input/update)
3. write_only (bool) — Prevents automatic output (useful for passwords)
4. required (bool) — Whether a field must be present during deserialization
5. allow_null (bool) — Allows null/None values
6. default — Fallback value when no input is provided
7. source — Field name mapping from the model/object
8. label — Custom human-readable label
9. validators — Custom validation callables
10. error_messages — Custom error message templates

e.g. :-> 

from rest_framework import serializers

class StudentSerializer(serializers.Serializer):
    # Field with custom arguments
    name = serializers.CharField(max_length=100, required=True, allow_null=False)
    
    # Read-only field (only output, no input)
    id = serializers.IntegerField(read_only=True) 
    
    # Write-only field (only input, no output) - used for passwords
    password = serializers.CharField(write_only=True) 
    
    # Field with a default value
    status = serializers.CharField(default="active")
    
    # Field with source (maps to a different model attribute)
    student_code = serializers.CharField(source="roll_number")
    
    # Field with custom validation
    def validate_age(self, value):
        if value < 18:
            raise serializers.ValidationError("Age must be at least 18")
        return value

    instance = Student.objects.get(id=validated_data.get('id'))
    instance.name = validated_data.get('name', instance.name)
    instance.age = validated_data.get('age', instance.age)
    instance.city = validated_data.get('city', instance.city)
    instance.save()
    return instance

3.   Validators >> It is used to validate multiple fields
                 >> we want to place our validation logic into resuable components, so that it can be reused throughout our codebase
                 >> Advantages : 1. It introduce proper sepration of concerns, making your code behaviour more obious
                 >>              2. it is easy to shift between ModelSerializer class and using explicit serializerclass
                 

                 e.g>>

                 from rest_framework import serializers

                 class StudentSerializer(serializers.Serializer):
                    name = serializers.CharField(max_length=100)
                    roll = serializers.IntegerField(max_length=3)
                    city = serializers.CharField(max_length=100)



                    def start_with_r(value):
                        if value['0'].lower() != 'r':
                            raise serializers.ValidationError("Name must start with r")
                        return value 

                    class StudentSerializer(serializers.Serializer):
                        name = serializers.CharField(max_length = 100, validators = [start_with_r]) 
                        roll = serializers.IntegerField(max_length = 3)
                        city = serializers.CharField(max_length = 100)

--->  ModelSerializer Class
1.  the ModelSerializer class provide a shortcut that lets you antomatically create a serializer class with files that correspond to the model fields

2. ModelSerializer class is the same as regular serializer class , insted of:
        * it will automatically generate set of fields for you, based on the model.
        * it will automatically generate validators for the serializer, such as unique_together validators
        * it include simple defult implementations of create() and update()


in serializer.py

from rest_framework import serializers 
from .models import Student 

class StudentSerializer(serializers.ModelSerializer):
    class Meta:
        model = Student
        fields = ['name', 'roll', 'city']


    # Note : If we not write Meta class then we have to write all the fields manually
    #        like : name = serializers.CharField(max_length=100), roll = serializers.IntegerField(), city = serializers.CharField(max_length=100)

    *fields= '__all__' ---> it will automatically generate set of files for you, based on the model.

    *exclude= ['roll']  ---> it will exclude the roll field from the serializer.

***********************Request************
in REST Framework Request object provide flexible request parsing that allows you to treat request with JSON Data  or another media data

1. request.data --> It helps to parse the request data like when we are posting JSON data
2. request.method --> it helps to identify the type of request like POST, GET, PUT, PATCH, DELETE
3. request.query_params --> It helps to get the query parameters from the request url like  ?search=xyz
4. request.user --> it access the user authentication details

***********************Response************
in REST Framework support HTTP content negotiation and automatically renders response objects into appropriate format.

1. Response() -> it is a response class that is used to return the response
2. status -> it is a status code that is used to return the status code like 200, 201, 400, 401, 404, 500
3. content -> it is a content that is used to return the content like json data
4. headers -> it is a headers that is used to return the headers like content-type, content-length
5. data -> it is a data that is used to return the data like json data
